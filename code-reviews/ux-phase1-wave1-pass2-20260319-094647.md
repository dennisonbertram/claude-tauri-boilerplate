**CRITICAL ISSUES**

1. **Empty state CTA handlers (e.g., onNewChat) May Be Undefined**
   - In both `SessionSidebar` and `WelcomeScreen` (and possibly other empty state screens), when the button is clicked, handlers like `onNewChat` are called directly (`onClick={onNewChat}`). There’s no null/undefined check or fallback. If these props are `undefined` or `null`, this will throw a runtime error.
   - **Severity: CRITICAL**

**HIGH ISSUES**

2. **StatusBar: PermissionMode Segment Hidden in "default" — User May Not Know Mode**
   - If permission mode is `'default'`, the PermissionMode segment is not rendered (returns `null`). There is no indicator whatsoever of permission mode in this base state. Users with a need to verify they're in default/normal mode have no way to confirm.
   - **Severity: HIGH**

3. **ProjectSidebar/SessionSidebar: UserBadge move to bottom may break position-based tests**
   - Moved from top to bottom. If tests or features expect UserBadge at the top, or query for it with top-first logic, those will fail. This can break existing automated UI tests or onboarding flows which depend on position.
   - **Severity: HIGH**

**MEDIUM ISSUES**

4. **Resource usage display (CPU/MEM combined) can break on small status bar widths**
   - The display is "CPU% / MEM MB", side by side. On a very small status bar/viewport, text can overflow or become unreadable or clipped, as there is no explicit truncation/ellipsis or responsive fallback.
   - **Severity: MEDIUM**

5. **Incorrect handler wiring — Empty state CTAs may call wrong/no handler**
   - From code review, CTA buttons for "Add Project", "New Team", "New Workspace", etc., are usually wired to correct handlers, but there's no guarantee and several are duplicated independently in sidebar and empty state (not always using the same handler). Risk of accidental mis-ref, especially if future refactors occur.
   - **Severity: MEDIUM**

**LOW ISSUES**

6. **StatusBar: Tooltip for CPU/MEM only visible if segment present**
   - Not user-breaking, but if the segment doesn't render, user can't mouse-over and see system stats via tooltip. Not a bug, but slight inconsistency.
   - **Severity: LOW**

7. **Dialog tabs (CreateWorkspaceDialog) now have icons, but no aria-labels**
   - New icons added, but accessible label is only the visible text label; if ever labels are removed for compactness, a11y could be impacted.
   - **Severity: LOW**

**Edge Cases/UX Notes**

- PermissionMode "default" case is a classic UX hidden state issue. If a user wants to know *which* permission mode they're in, and sees nothing, they may assume they're in an unusual/unintended mode. To solve: Even for 'default', show as "Normal" or similar with a tooltip, grayed out.
- Empty state CTA error (undefined handler) risk is high in safety-critical apps. Always guard onClick to ensure you don't throw on undefined.
- For resource usage, should at least clamp or shrink font on tiny widths, or collapse to an icon+tooltip if possible.
