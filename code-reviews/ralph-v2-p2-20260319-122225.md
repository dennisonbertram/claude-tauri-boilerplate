## CRITICAL

1) **Canvas does not re-initialize when `hooksJson` / `hooksCanvasJson` props change (stale canvas ‚Üí data corruption)**
- **Where:** `HookCanvasInner` mount effect has `[]` deps and is guarded by `initialized.current`.
- **How to break:** Open Canvas view for Agent A, then switch to Agent B (or otherwise update `draft.hooksJson` while staying on Canvas). The canvas will still show A‚Äôs nodes; any edit will call `onChange(...)` and overwrite Agent B‚Äôs hooks with Agent A-derived compilation.
- **Fix:** Add a prop-change reconciliation path:
  - Reset `initialized.current` when props change, or
  - Key the `<HookCanvas />` by agent/profile id so it remounts, or
  - Implement an effect watching `hooksJson/hooksCanvasJson` and performing a safe reload (with ‚Äúunsaved changes‚Äù handling).

2) **`generateCanvasFromHooks` can OOM/freeze the UI before truncation**
- **Where:** `canvasGenerator.ts` builds `nodes`/`edges` for all hooks, and only truncates *after* full construction.
- **How to break:** Paste (or import) a hooks JSON with many events/groups/hooks (even within a few MB), e.g. thousands of hooks; it will allocate huge arrays and create many objects before it ever hits the truncation logic.
- **Fix:** Enforce caps *during generation* (early breaks) and/or estimate counts before allocation. Example: stop generating once `nodes.length >= MAX_GENERATED_NODES` or `edges.length >= MAX_EDGES`.

3) **Unsupported hook types/events get silently dropped when editing in Canvas (destructive lossy transform)**
- **Where:** `buildActionData()` returns `null` for unrecognized `hook.type`, so those hooks never appear on canvas; later `compileCanvasToHooks()` outputs only what‚Äôs representable.
- **How to break:** Start with a profile that includes a hook type you don‚Äôt support yet. Switch to Canvas and make any edit that triggers compilation (e.g., edit a matcher). The resulting `hooksJson` overwrites the original and **removes** those unsupported hooks with no warning.
- **Fix:** Either:
  - Render ‚ÄúUnsupportedActionNode‚Äù placeholders that round-trip the raw hook entry, or
  - Block compilation when unsupported entries were detected (show a banner and force JSON editing), or
  - Preserve original JSON and merge only the representable subset (harder, but avoids data loss).

## HIGH

1) **Node layout/positions are not reliably persisted (drag-only changes are lost)**
- **Where:** Fingerprint excludes `position` to avoid recompiling hooks; `notifyChange()` isn‚Äôt called on drag unless some other structural/data change happens later.
- **User impact:** Reposition nodes, close/reopen ‚Üí layout snaps back to the last time something else changed.
- **Fix:** Split persistence:
  - Save `hooksCanvasJson` on position changes (possibly debounced), **without** regenerating `hooksJson`.
  - Only regenerate `hooksJson` when semantic structure changes.

2) **Race condition: ‚Äúsingle inbound edge per action node‚Äù enforcement can be bypassed**
- **Where:** `onConnect` checks `edges.some(...)` using the closure-captured `edges`, then calls `setEdges(addEdge...)`.
- **How to break:** Rapidly connect multiple parents to the same action (or multi-pointer / fast repeated actions). State updates can interleave and allow duplicates.
- **Impact:** Canvas shows multiple inbound edges, but compiler‚Äôs `processedActionIds` will silently drop duplicates ‚Üí mismatch between visual graph and emitted JSON.
- **Fix:** Perform validation inside the functional updater:
  ```ts
  setEdges((eds) => {
    if (targetType === 'action' && eds.some(e => e.target === params.target)) return eds;
    if (eds.length >= MAX_EDGES) return eds;
    return addEdge(params, eds);
  });
  ```

3) **Severe performance risk: fingerprinting does `JSON.stringify` of all node `data` on every nodes/edges change**
- **Where:** `getHooksFingerprint(nodes, edges)` stringifies full `data` blobs; effect runs whenever `nodes` changes (including each keystroke in config panel).
- **How to break:** Add a few large prompt nodes (tens of KB each) and type; UI will lag/hang due to repeated full serialization.
- **Fix:** Replace with incremental hashing / versioning:
  - Track a ‚Äúdirty semantic version‚Äù per node id, or
  - Hash only stable minimal fields + per-node revision counters, or
  - Use `fast-json-stable-stringify` on a reduced subset, or
  - Debounce *fingerprint computation* too.

4) **Trigger event select cannot represent unknown/custom events (editing forces data loss)**
- **Where:** `TriggerConfig` uses `<select>` with only `HOOK_EVENTS`. If `data.event` isn‚Äôt in the list, it becomes effectively uneditable without overwriting.
- **Fix:** Allow a freeform input (or a combobox) that supports custom events, and/or include the current value as an option if missing.

5) **Any JSON typing immediately clears `hooksCanvasJson` (layout loss with no confirmation)**
- **Where:** HooksTab JSON textarea `onChange` does `hooksCanvasJson: null`.
- **How to break:** User spends time arranging canvas ‚Üí switches to JSON to copy/inspect ‚Üí hits space/backspace ‚Üí canvas layout is discarded permanently.
- **Fix:** Don‚Äôt eagerly null it; instead:
  - Mark it ‚Äúout of sync‚Äù and prompt before discarding, or
  - Keep canvas json until user explicitly chooses ‚ÄúRegenerate canvas from JSON‚Äù.

6) **Untrusted `hooksCanvasJson` is accepted with minimal sanitization (can embed huge strings/objects leading to perf/memory issues)**
- **Where:** `isValidCanvasState` checks shape and counts, but does not sanitize `node.data` size/keys, and `notifyChange` serializes `currentNodes` back to JSON.
- **Fix:** Sanitize loaded `node.data` similarly to `sanitizeNodeData`, plus enforce per-field size caps (especially prompt/command).

## MEDIUM

1) **Dangerous-hook warning heuristic is brittle in `hasDangerousHooks`**
- **Where:** `HooksTab` checks raw string `includes('"type":"command"')` with a couple spacing variants; misses newlines or other spacing.
- **Fix:** Parse JSON when valid and walk structure looking for `type === 'command'|'http'`.

2) **Import failures are silently ignored**
- **Where:** `handleImport` catch block is empty (`// silently ignore`).
- **Impact:** User selects a file and nothing happens; no remediation path.
- **Fix:** Show a toast/modal with the parse error; also consider schema validation errors.

3) **`URL.revokeObjectURL(url)` immediately after `.click()` can break downloads in some browsers**
- **Fix:** Revoke in a `setTimeout(..., 0/1000)` or on `a.onload`-equivalent pattern.

4) **`crypto.randomUUID()` may be unavailable in some environments**
- **Where:** `onDrop` node id generation.
- **Fix:** Fallback to a small uuid polyfill or `Date.now()+Math.random()`.

5) **Canvas gives no UX feedback when a connection is rejected**
- **Where:** `onConnect` just returns on invalid; `getInvalidConnectionReason` exists but is unused.
- **Fix:** Show toast with reason, and/or use ReactFlow `isValidConnection` to prevent invalid target highlighting.

6) **Numeric inputs allow NaN/Infinity edge cases**
- **Where:** `Number(e.target.value)` without clamping; can store `Infinity`, `NaN`.
- **Fix:** Validate and clamp at input time; normalize to `undefined` if not finite.

- **Where:** `icon="&#9889;"` etc passed as string ‚Üí rendered as text.
- **Fix:** Use actual unicode characters (`"‚ö°"`, `"‚öô"`) or render as JSX.
