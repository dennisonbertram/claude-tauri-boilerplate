## CRITICAL

### 1) Cross-profile data corruption via debounced save `flush()` on unmount
**Where:** `src/components/agent-builder/HookCanvas.tsx` (unmount effect calls `saveCanvas.flush()`), `src/components/agent-builder/tabs/HooksTab.tsx` (canvas keyed by `profileId`)

**What happens:** If a user edits Profile A in Canvas, then quickly switches to Profile B (changing `profileId`, causing the old `HookCanvas` to unmount), the unmount cleanup **flushes pending edits** and calls `onChange(...)`. But `onChange` has **no profile scoping**, so the parent may apply those updates to the *currently active profile* (now Profile B). This is a classic “late write from previous instance” race.

**How to reproduce:**
1. Open Profile A → Canvas.
2. Drag nodes / edit fields (ensures a pending debounced save).
3. Immediately switch to Profile B within 500ms.
4. Observe Profile B’s hooks JSON/canvas JSON may get overwritten by Profile A’s flush.

**Impact:** Silent cross-profile overwrite (data integrity loss). Potentially catastrophic if command/http hooks get copied into the wrong profile.

**Fix:**
- Include `profileId` (or a `scopeKey`) in the `onChange` contract: `onChange(profileId, hooksJson, hooksCanvasJson)` and ignore mismatched updates in the parent.
- Alternatively: in `HookCanvas`, capture an `instanceId` ref and only call `onChange` if still current; or **cancel** pending debounced writes on unmount instead of flushing, and require an explicit “Apply/Save” before switching profiles.


## HIGH

### 2) Canvas load can persist a hooks/canvas mismatch (layout-only edits keep stale `hooksJson`)
**Where:** `HookCanvas.tsx` initialization when `hooksCanvasJson` exists:
```ts
lastCompiledHooksJsonRef.current = hooksJson ?? compileCanvasToHooks(...)
```
Then later, for non-structural changes:
```ts
hooksJson = lastCompiledHooksJsonRef.current || compileCanvasToHooks(...)
```

**What happens:** If `hooksCanvasJson` and `hooksJson` ever diverge (external edits, partial restores, migrations, bugs), the canvas will load from `hooksCanvasJson` but **continue emitting the stale `hooksJson`** for layout-only changes (moving nodes), because `structuralDirtyRef` stays false. That means the app can repeatedly save an inconsistent pair: `{hooksJson: old, hooksCanvasJson: new}`.

**Impact:** Persistent divergence, confusing “canvas doesn’t match JSON”, and potential later destructive overwrite when a structural change finally triggers compilation.

**Fix:** When loading from `hooksCanvasJson`, set `lastCompiledHooksJsonRef.current` to the **compiled** output from loaded nodes/edges (or at least detect mismatch and force a one-time recompile + `onChange` sync).


### 3) Node limit can be exceeded due to stale `nodes.length` check outside the state updater
**Where:** `HookCanvas.tsx` `onDrop`:
```ts
if (nodes.length >= MAX_NODE_COUNT) return;
setNodes((nds) => [...nds, newNode]);
```

**What happens:** The limit check uses `nodes` from the render closure, not the latest `nds` inside the functional update. Rapid drops (or concurrent React updates) can bypass the limit.

**Impact:** Exceeding intended caps can degrade performance and bloat saved JSON.

**Fix:** Enforce the limit inside the updater:
```ts
setNodes(nds => nds.length >= MAX_NODE_COUNT ? nds : [...nds, newNode]);
```


### 4) Condition with empty matcher compiles to “unconditional” group (surprising semantics)
**Where:** `canvasCompiler.ts`:
```ts
if (condData.matcher) group.matcher = condData.matcher
```
Empty string → matcher omitted → group becomes unconditional.

**What happens:** A user can add a Condition node and leave matcher blank (“No matcher” in UI), expecting “this condition is not configured yet / matches nothing”. Instead, it compiles as **no matcher**, which commonly means “apply to all tools” (unconditional), potentially executing attached command/http hooks unexpectedly.

**Impact:** High-risk behavioral footgun, especially with dangerous actions.

**Fix options:**
- Treat empty matcher as invalid and **skip** the group (or mark group as disabled) until matcher is non-empty.
- Or explicitly compile empty matcher as a matcher that matches nothing (if the hooks format supports it), but don’t silently convert it to unconditional behavior.
- UI should clearly label empty matcher as “Unconditional” if that’s intended.


## MEDIUM

### 5) Incomplete action nodes remain visible but compile away (canvas ≠ JSON)
**Where:** `canvasCompiler.ts` `buildHookEntry()` returns `null` unless required fields are non-empty (`command`, `url`, `prompt`, `description`).

**What happens:** You can have action nodes on the canvas (including saved canvas JSON) that silently disappear from compiled `hooksJson`. The user can believe “it’s in my flow” while the exported JSON omits it.

**Impact:** Confusing, easy to ship a profile missing expected hooks; round-tripping between JSON/canvas can “lose” nodes.

**Fix:** Show validation errors on nodes/panel (“This action won’t be included until you set URL/command/etc”), and/or prevent saving/compiling without required fields (or compile placeholders explicitly if supported).


### 6) `isValidCanvasState` allows `data` to be an array (and other odd objects)
**Where:** `HookCanvas.tsx`:
```ts
if (!n.data || typeof n.data !== 'object') return false;
```
Arrays pass `typeof === 'object'`.

**What happens:** Malformed saved canvas JSON can pass validation with `data: []`. Downstream components assume object fields (`data.event`, `data.matcher`), causing uncontrolled inputs, blank configs, or inability to edit.

**Fix:** Reject arrays explicitly:
```ts
if (!n.data || typeof n.data !== 'object' || Array.isArray(n.data)) return false;
```
Also validate required fields like `action.data.hookType` is a supported string.


### 7) No size cap for `hooksCanvasJson` parsing (possible UI freeze)
**Where:** `HookCanvas.tsx` mount effect parses `hooksCanvasJson` with `JSON.parse` without size checks.

**Impact:** A huge stored canvas blob (or corrupted state) can freeze the UI thread.

**Fix:** Cap `hooksCanvasJson.length` (or approximate bytes) before parsing; fail closed and regenerate from hooks JSON.


### 8) Debounced save function is recreated if `onChange` identity changes → out-of-order writes
**Where:** `HookCanvas.tsx`:
```ts
const saveCanvas = useCallback(debounce(() => { ... onChange(...) }, 500), [onChange]);
```

**What happens:** If the parent passes a non-memoized `onChange`, `saveCanvas` is recreated often. Old timers can still fire (or get flushed in effect cleanup) and call older `onChange` closures, producing out-of-order saves.

**Fix:** Store `onChange` in a ref (`onChangeRef.current = onChange`) and create the debounced function once (empty deps), reading from the ref at execution time. Also cancel timers on replacement.


### 9) Trigger event select can become “invalid value” if event is missing/unknown
**Where:** `NodeConfigPanel.tsx` `TriggerConfig`:
```tsx
<select value={data.event}> ... {HOOK_EVENTS.map(...)}
```

If `data.event` is `undefined` (bad state) or an unknown event key (allowed by generator), the select won’t have a matching option and can display blank / warn.

**Fix:** Provide a fallback option or render unknown values as an `<option>` so they remain editable without being silently coerced.


### 10) URL CRLF is allowed in canvas state but later causes hook drop
**Where:** `sanitizeNodeData` doesn’t strip CRLF for `url`; compiler rejects CRLF URL and returns `null` hook.

**Impact:** Node appears configured, but compiled hook disappears; confusing “why didn’t my HTTP hook export?”

**Fix:** Validate/strip CRLF at edit time (config panel / sanitize step) and show an error state instead of silently dropping.


## LOW

### 11) Misleading warning: declining dangerous canvas load logs “Invalid canvas state”
**Where:** `HookCanvas.tsx` — when user clicks “Cancel” in confirm, it falls through and prints:
```ts
console.warn('[HookCanvas] Invalid canvas state, regenerating...')
```
even though it was valid.

**Fix:** Separate “user declined” from “invalid state” code paths.


### 12) Delete key handling likely incomplete across platforms
**Where:** `HookCanvas.tsx`:
```tsx
deleteKeyCode="Delete"
```
Some environments use `Backspace`.

**Fix:** Support both (array) or rely on ReactFlow defaults.


### 13) Import invalid JSON fails silently
**Where:** `HooksTab.tsx` `handleImport` catch block does nothing.

**Impact:** Confusing UX (“nothing happened”).

**Fix:** Show an error toast/dialog.


### 14) Export revokes object URL immediately (Safari risk)
**Where:** `HooksTab.tsx`:
```ts
a.click();
URL.revokeObjectURL(url);
```
Some browsers need revocation delayed.

**Fix:** `setTimeout(() => URL.revokeObjectURL(url), 0/1000)`.
