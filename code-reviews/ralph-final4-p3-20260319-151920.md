## HIGH

1) **Hooks JSON can become stale / inconsistent with the loaded canvas state (especially after sanitization or edge pruning)**
- **Where:** `src/components/agent-builder/HookCanvas.tsx`
- **What happens:**
  - When loading `hooksCanvasJson`, the code sanitizes node data and **prunes invalid edges** (`pruneInvalidEdges(...)`), then sets state with the sanitized/pruned graph.
  - But it sets `lastCompiledHooksJsonRef.current` to `hooksJson ?? compileCanvasToHooks(...)`, meaning **if `hooksJson` exists it will be treated as authoritative even though the canvas was just mutated** (sanitized/pruned).
  - On the subsequent autosave, `saveCanvas()` will often emit `hooksJson = lastCompiledHooksJsonRef.current` because `structuralDirtyRef.current` is still `false`.
- **Why this breaks invariants:** In canvas mode, the invariant should be ‚Äúrendered canvas graph === serialized hooks JSON‚Äù. Automatic mutation (sanitization/pruning) is a semantic change, but it does **not** force recompilation, so the UI can show a different flow than what `hooksJson` actually contains.
- **Impact:**
  - Hooks can remain in `hooksJson` that are no longer represented by the canvas (e.g., edges pruned away).
  - Sanitization/limits applied to the canvas may **not** apply to the effective `hooksJson` unless the user performs a ‚Äústructural‚Äù edit later.
- **Fix:**
  - When you load and mutate the graph (sanitize/prune), treat that as dirty:
    - Either set `structuralDirtyRef.current = true` after loading sanitized nodes/edges, **or**
    - Set `lastCompiledHooksJsonRef.current = compileCanvasToHooks(sanitizedNodes, prunedEdges)` unconditionally when canvas JSON is the source of truth.
  - Consider comparing provided `hooksJson` to compiled output and forcing sync if they differ.

## MEDIUM

2) **Canvas round-trip is not shape-preserving for ‚Äúno matcher‚Äù groups**
- **Where:** `src/lib/canvasCompiler.ts` vs `src/lib/canvasGenerator.ts`
- **What happens:**
  - Generator supports a group with `matcher` omitted and multiple `hooks` (it will create multiple trigger‚Üíaction edges).
  - Compiler, for trigger‚Üíaction directly, emits **one group per action**: `groups.push({ hooks: [hookEntry] })`.
- **Impact:** `hooks.json -> canvas -> hooks.json` changes structure (splits groups), which can cause noisy diffs and can become semantically risky if group-level semantics expand in the future (or if ordering/grouping is meaningful externally).
- **Fix:** For trigger‚Üídirect actions, consider emitting a single group with `hooks: [...]` (preserve the original grouping), or represent ‚Äúgroup‚Äù explicitly in the canvas model.

3) **Unknown hook events are ‚Äúpreserved‚Äù by compiler but not safely editable in UI**
- **Where:** `NodeConfigPanel.tsx` (trigger event `<select>` only lists `HOOK_EVENTS`), `canvasCompiler.ts` (warns but ‚Äúpreserves unknown events‚Äù)
- **Impact:**
  - If JSON contains an event not in `HOOK_EVENTS`, the canvas can generate a trigger node with that event, but the config UI cannot represent it as an option.
  - Any user interaction that changes the select will force a supported event, effectively losing the unknown event.
- **Fix:** Include the current event value as an `<option>` when it‚Äôs not in `HOOK_EVENTS`, or switch to a combo-box/free-text event input with validation.

4) **Connection rules are enforced on `onConnect` and load-pruning, but not guaranteed for other edge mutations**
- **Where:** `HookCanvas.tsx`
- **What happens:** You validate in `onConnect(...)`, but React Flow can mutate edges via other mechanisms (edge update/reconnect behaviors, programmatic changes, future UI features). `handleEdgesChange` marks dirty but does not revalidate edges.
- **Impact:** Invalid edges can enter state and be compiled in unexpected ways (or later dropped), breaking the ‚Äúgraph validity‚Äù invariant.
- **Fix:**
  - Pass ReactFlow‚Äôs `isValidConnection` prop (using `checkConnection`) to block invalid connections at the framework level.
  - Additionally, consider validating/pruning edges inside `handleEdgesChange` (or before compile) to enforce invariants like ‚Äúsingle inbound edge per action‚Äù.

## LOW

5) **Non-deterministic output ordering**
- **Where:** `canvasCompiler.ts` adjacency traversal uses raw edge insertion order; node/edge arrays may not be stable.
- **Impact:** JSON diffs can churn between saves even if semantics unchanged.
- **Fix:** Sort connected nodes (e.g., by node `position.y`, then id) before emitting hooks.

6) **`debounce.flush()` generic behavior is unsafe (though current usage is 0-arg)**
- **Where:** `HookCanvas.tsx` debounce implementation
- **Impact:** If reused with arg-taking functions later, `flush()` may call `fn()` without args when `lastArgs` is null.
- **Fix:** Restrict debounce utility to zero-arg functions here, or require args to exist before flushing.

7) **`sanitizeNodeData` retains `undefined` keys**
- **Where:** `HookCanvas.tsx`
- **Impact:** Minor; can create subtle differences between in-memory state vs JSON-serialized state (JSON drops `undefined`).
- **Fix:** Skip assigning `undefined` values entirely.
