## CRITICAL

1) **State mismatch can overwrite hooks JSON with stale content (hooksJson vs hooksCanvasJson)**
- **Where:** `HookCanvasInner` mount effect sets `lastCompiledHooksJsonRef.current = hooksJson ?? compileCanvasToHooks(...)` when loading **saved canvas**.
- **Failure mode:** If `hooksCanvasJson` is present but `hooksJson` is also present (and they disagree), the canvas will load from `hooksCanvasJson`, but **layout-only edits** (dragging) call `saveLayout()` which persists `lastCompiledHooksJsonRef.current` (potentially the *stale* `hooksJson`) alongside the new canvas layout. Net: you can silently ‚Äúrevert‚Äù hooks.
- **How to break:** Arrange persisted profile where `hooksCanvasJson` reflects newer canvas than `hooksJson` (common after schema changes, partial migrations, external edits, sync bugs). Open canvas, move a node, wait 2s ‚Üí `onChange(oldHooksJson, newCanvasJson)` persists the old JSON.
- **Fix:** When loading from `hooksCanvasJson`, **always** set `lastCompiledHooksJsonRef` from `compileCanvasToHooks(loadedCanvas)` (or assert equivalence and choose a single source of truth; ideally: hooksJson is canonical, canvasJson is layout-only).

2) **Debounced saves are canceled on unmount ‚Üí easy data loss when switching views quickly**
- **Where:** `notifyChange` (500ms) and `saveLayout` (2000ms) are `.cancel()`ed on unmount; no flush.
- **Failure mode:** Add node / connect edge / edit config ‚Üí immediately switch Canvas‚ÜíJSON (or navigate away) within debounce window ‚Üí changes never reach parent `draft` state.
- **How to break:** Drop a node and quickly toggle view; the canvas state disappears because parent never received updated `hooksJson/hooksCanvasJson`.
- **Fix:** On unmount, **flush** pending debounces (run immediately) instead of canceling, or keep canvas state in parent as source-of-truth.

3) **Hooks can be silently dropped when entering Canvas (missing `type`) with no ‚Äúunsupported‚Äù warning**
- **Where:** `HooksTab.hasUnsupportedHooks` only flags hooks with `hook.type && !SUPPORTED`, but **does not flag hooks with no `type`**. `generateCanvasFromHooks()` drops them (`buildActionData` requires a valid `type`).
- **Failure mode:** A hook object `{ command: "..." }` (or `{ url: "..." }`) is treated as ‚Äúdangerous‚Äù by `hasDangerousHooks`, but **not** treated as ‚Äúunsupported‚Äù; switching to Canvas and any subsequent save will remove it permanently.
- **Fix:** Treat hooks with missing/invalid `type` as unsupported (block/confirm), or preserve them via ‚Äúraw/unknown hook‚Äù nodes, or keep them in JSON-only storage while editing canvas.

## HIGH

1) **Prototype-pollution surface via HTTP headers (compiler + generator inconsistencies)**
- **Where:**  
  - `canvasGenerator.buildActionData()` passes `hook.headers` through **without** stripping dangerous keys / CRLF / size limits.  
  - `canvasCompiler.buildHookEntry()` sanitizes CRLF/length but **does not** block `__proto__`, `constructor`, etc for header keys.
- **Failure mode:** A crafted `hooksJson` containing headers like `{"__proto__": {"polluted": true}}` can survive round-trips and later be consumed by other parts of the app/runtime that do `Object.assign` / spreads, potentially polluting prototypes.
- **Fix:** Sanitize header keys in **both** generator and compiler; prefer `Object.create(null)` for header maps; explicitly reject dangerous keys.

2) **Imported canvas state can contain invalid edges that UI would never allow; compiler output diverges from what user sees**
- **Where:** `isValidCanvasState()` only checks edge endpoints exist; it does **not** validate edge direction/type pairing via `connectionRules`, nor enforce ‚Äúaction has max 1 incoming edge‚Äù.
- **Failure mode:** Load a saved `hooksCanvasJson` with action‚Üícondition edges, multiple incoming edges to action, condition‚Üícondition, etc. ReactFlow renders edges; compiler largely ignores invalid structures (only trigger‚Üícondition/action, condition‚Üíaction), causing ‚Äúwhy didn‚Äôt my hook run?‚Äù confusion and data loss on save.
- **Fix:** Validate edges against `isValidConnection(sourceType,targetType)` and enforce action-incoming constraints during load (drop or repair invalid edges).

3) **NodeConfigPanel updates bypass sanitization ‚Üí Infinity/NaN + huge strings can corrupt persisted canvas JSON**
- **Where:** `onUpdateNode` merges `newData` directly into node `data` with no `sanitizeNodeData`.
- **Failure modes:**
  - Timeout can become `Infinity`/`NaN` (e.g., `1e309`) ‚Üí `JSON.stringify` turns it into `null`, creating hard-to-debug state drift.
  - Very large prompt/command pasted via config panel is not capped until compilation; but `hooksCanvasJson` will still store the huge string, risking storage/perf issues.
- **Fix:** Apply `sanitizeNodeData()` (and numeric clamping) inside `onUpdateNode` before storing.

4) **Unknown/custom trigger events are not representable in the Trigger select ‚Üí accidental mutation**
- **Where:** `NodeConfigPanel.TriggerConfig` renders `<select value={data.event}>` with options only from `HOOK_EVENTS`.
- **Failure mode:** If `hooksJson` contains a custom event key, generator will create a trigger node with `event="CustomEvent"`. The select will show blank/odd state; any user interaction can switch it to the first supported event, silently rewriting the event key on next save.
- **Fix:** Allow free-text event editing or include an ‚Äú(custom)‚Äù option that preserves the current value.

## MEDIUM

1) **‚ÄúAtomic load‚Äù isn‚Äôt truly atomic (setNodes + setEdges separately)**
- **Where:** mount effect calls `setNodes(...)` then `setEdges(...)`.
- **Risk:** In environments without guaranteed batching, you can briefly render nodes without edges, triggering structural watchers and possibly spurious saves/compiles.
- **Fix:** Batch updates explicitly (or store a single combined state / use batching utility).

2) **Drag-and-drop payload can create uneditable/invalid nodes (hookType not validated)**
- **Where:** `onDrop` only validates `nodeType` and ‚Äúdata is object‚Äù; doesn‚Äôt validate `data.hookType` for action nodes.
- **Failure mode:** Inject `hookType: "weird"` via crafted drag payload ‚Üí node exists, config panel shows no editor, compiler drops it.
- **Fix:** Validate `hookType` against supported set on drop; otherwise reject or coerce.

3) **Fingerprint includes `label` even though compilation ignores it ‚Üí unnecessary recompiles/churn**
- **Where:** `getHooksFingerprint()` includes full `data`; compiler ignores `label`.
- **Impact:** Editing fields that only affect labels (or auto-label updates) triggers recompile + parent `onChange`, increasing churn and risk of overwriting formatting.
- **Fix:** Exclude non-semantic fields (like `label`) from fingerprint, or make compiler depend on them intentionally.

4) **Generator doesn‚Äôt enforce header constraints (size/CRLF/danger keys)**
- **Where:** `buildActionData` assigns `headers` verbatim.
- **Impact:** Huge headers objects can bloat canvas state and slow down ReactFlow; CRLF headers can later be rejected inconsistently by compiler/runtime.
- **Fix:** Apply the same header sanitization in generator as in canvas sanitization/compiler.

5) **No user-facing indication of nodes that will be skipped by compiler**
- **Where:** `compileCanvasToHooks` silently skips incomplete actions (empty command/url/prompt/description) and triggers with missing event.
- **Impact:** Canvas can look ‚Äúconnected‚Äù but produce empty hooks JSON; users won‚Äôt know why until runtime.
- **Fix:** Visual validation (node warnings), or block save/compile with errors list.

## LOW

1) **NodePalette uses HTML entities as strings ‚Üí icons likely render literally**
- **Where:** `icon="&#9889;"` etc passed as string and rendered as `{icon}`.
- **Impact:** Cosmetic/UX confusion.
- **Fix:** Use the actual unicode character (e.g., `"‚ö°"`) or render entities in JSX, not strings.

2) **Invalid JSON import fails silently**
- **Where:** `handleImport` catch block does nothing.
- **Impact:** Users think import worked; leads to confusion.
- **Fix:** Show an error toast/dialog.

3) **`crypto.randomUUID()` may not exist in some runtimes**
- **Impact:** Drop-node fails hard in older webviews.
- **Fix:** Fallback to a UUID polyfill.
