## Audit: Visual Hook Builder (React Flow canvas ‚áÑ Claude Code hooks JSON)

### Invariants & state transitions (what the code is trying to guarantee)
- **Graph shape invariants**
  - Valid edge directions enforced at connect-time via `connectionRules.isValidConnection`: `trigger ‚Üí condition|action`, `condition ‚Üí action`, `action ‚Üí (none)`.
  - **Single inbound edge per action** enforced in two places:
    - During interaction: `onConnect` rejects if an action already has any inbound edge.
    - During load: `pruneInvalidEdges` drops extra inbound edges to actions.
- **Persistence invariants**
  - Canvas state saved as `{nodes, edges}` JSON; hooks JSON compiled from canvas when ‚Äústructural‚Äù changes occur.
  - ‚ÄúLayout-only‚Äù changes (position dragging) are intended to **not** recompile hooks JSON (uses `structuralDirtyRef` + `lastCompiledHooksJsonRef`).
- **Input hardening**
  - Multiple layers of sanitation/validation: `isValidCanvasState` (shape/type), `stripDangerousKeys` + `sanitizeNodeData` (length limits, CRLF stripping in headers/URL, prototype-pollution keys blocked), compiler/generator also sanitize.

Those are generally coherent and mostly enforced consistently.

## Findings

### HIGH ‚Äî Unsaved edits can be lost on unmount due to `saveCanvas.cancel()`
**Where:** `HookCanvas.tsx` unmount cleanup:
```ts
useEffect(() => {
  return () => { saveCanvas.cancel(); };
}, [saveCanvas]);
```
**Why it matters:** `saveCanvas` is debounced at 500ms. If the user:
- edits nodes/edges, then
- quickly switches from **Canvas ‚Üí JSON** view (unmounts `HookCanvas`) within 500ms,

‚Ä¶the pending save is **canceled**, so `onChange(hooksJson, hooksCanvasJson)` never runs and the parent draft does not receive the latest edits. This is a real state-transition correctness bug (lost updates).

**Fix:** On unmount, prefer flushing instead of canceling (or do both safely):
- `saveCanvas.flush()` (then `cancel()`), or
- immediate non-debounced ‚Äúfinal save‚Äù using latest refs, or
- shorten debounce and/or save on view-toggle before unmount.

### MEDIUM ‚Äî Conditions can be shared across triggers, producing duplicated/ambiguous compiled output
**Where:**
- `connectionRules.ts` allows multiple triggers connecting to the same condition (no restriction).
- `compileCanvasToHooks()` treats each trigger independently; for a shared condition node, its actions will be compiled under *each* trigger‚Äôs event.

**Effect:** A user can accidentally create a graph where a single condition/action subtree is reachable from multiple triggers. The compiler will duplicate hook groups across events. This is likely not intended UX/semantics.

**Fix options:**
- Enforce **single inbound edge per condition** (like actions) either in `onConnect` and `pruneInvalidEdges`, or
- Change compiler semantics to detect shared subgraphs and either error/warn or choose a deterministic ‚Äúowner‚Äù trigger.

### MEDIUM ‚Äî Empty matcher condition nodes don‚Äôt round-trip (canvas ‚Üí hooks ‚Üí canvas)
**Where:**
- Compiler: for `trigger ‚Üí condition ‚Üí action`, it creates a group even if `condData.matcher` is falsy, but omits `matcher`.
- Generator: it only creates a condition node if `group.matcher` is truthy:
```ts
if ((group as HookGroupInput).matcher) { /* create condition */ } else { /* direct */ }
```

**Effect:** A condition node with an empty matcher (common while editing) compiles to a no-matcher group; regenerating from hooks JSON removes the condition node entirely. This breaks structural invariants users may expect (graph shape not preserved).

**Fix:**
- Disallow/auto-remove empty-matcher conditions in the canvas (UI validation), or
- Preserve explicit empty matcher (e.g., `matcher: ""`) and treat it consistently in generator, or
- Compiler could skip condition nodes with empty matcher and instead behave as trigger‚Üíactions directly (to match regeneration), but that still changes connectivity.

### LOW ‚Äî NodePalette icons render as literal HTML entities
**Where:** `NodePalette.tsx` passes strings like `"&#9889;"` to a `<span>{icon}</span>`, which renders the literal characters rather than the symbol.

**Fix:** Use the actual unicode character (`"‚ö°"`, `"‚öô"`) or render via `dangerouslySetInnerHTML` (unicode preferred).

### LOW ‚Äî Type soundness gaps in validation for action `hookType` and some fields
**Where:**
- `isValidCanvasState()` validates some action fields (`command/url/prompt/description`) but not `hookType` being a supported string union.
- Runtime code frequently casts `as any` / `as CanvasNode`.

**Effect:** Malformed `hooksCanvasJson` can load an action node with an invalid `hookType`, leading to silent dropping in compilation (since `buildHookEntry` returns `null`) and confusing UI fallbacks.

**Fix:** Extend `isValidCanvasState` to validate `action.data.hookType` ‚àà `{command,http,prompt,agent}` and ensure expected field types per hookType.
