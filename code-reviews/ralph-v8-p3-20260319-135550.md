## Audit focus: invariants, state transitions, compilation logic, type soundness (Visual Hook Builder: React Flow canvas → hooks JSON)

### HIGH

1) **`hooksJson` ↔ `hooksCanvasJson` can become desynchronized on load (layout-only saves can persist stale hooks JSON)**
- **Where:** `HookCanvasInner` mount effect (“Priority 1: saved canvas JSON”).
- **What happens now:**
  - If `hooksCanvasJson` exists and is valid, the canvas state is loaded (nodes/edges are set).
  - But `lastCompiledHooksJsonRef.current` is set to `hooksJson ?? compileCanvasToHooks(...)`.
  - If `hooksJson` is non-null but **does not actually match** the loaded canvas (possible due to prior bugs, manual edits, migrations, or partial writes), then:
    - **Position-only drags** trigger `saveLayout()`, which writes `hooksCanvasJson` using the **stale** `lastCompiledHooksJsonRef.current` hooks JSON.
    - This can persist an inconsistent pair (`hooksJson` for one graph, `hooksCanvasJson` for another) without ever recompiling.
- **Invariant violated:** “Canvas JSON and hooks JSON represent the same semantics.”
- **Fix recommendation:**
  - When loading from `hooksCanvasJson`, always compute and store:
    - `const compiled = compileCanvasToHooks(sanitizedNodes, sanitizedEdges);`
    - `lastCompiledHooksJsonRef.current = compiled;`
  - Optionally, if `hooksJson` prop is present and differs from `compiled`, consider syncing via `onChange(compiled, hooksCanvasJsonNormalized)` (immediately or via a guarded one-shot).

---

### MEDIUM

2) **Loaded canvas state validation does not enforce connection invariants (rules + single-incoming action)**
- **Where:** `isValidCanvasState()` only checks existence/types + dangling edges.
- **Missing checks:**
  - Edge validity against `connectionRules` (`trigger→condition|action`, `condition→action`, no others).
  - “Action has at most one incoming edge” invariant (enforced during interactive `onConnect`, but **not** on load).
- **Impact:**
  - Malformed/older canvas JSON can load into an illegal graph that compiles unpredictably (e.g., action reachable from multiple triggers/conditions).
- **Fix recommendation:** During `isValidCanvasState` (or immediately after parsing), reject or filter edges that:
  - violate `isValidConnection(sourceType, targetType)`
  - target an action that already has an incoming edge (choose first edge deterministically).

3) **Sanitization gaps: generated canvas from `hooksJson` is not passed through the same sanitation as `hooksCanvasJson`**
- **Where:** mount effect “Priority 2: generate from hooks JSON” directly `setNodes(generated.nodes)` / `setEdges(generated.edges)`.
- **Why it matters:**
  - `sanitizeNodeData/stripDangerousKeys` are applied to loaded `hooksCanvasJson`, but **not** to nodes generated from `hooksJson`.
  - Today strings are sliced in `buildActionData`, but:
    - event key/label length is not bounded
    - `headers` are accepted wholesale (no dangerous-key removal, no CRLF/size filtering at this stage)
- **Impact:** inconsistent guarantees depending on entry path; possible UI/perf issues from pathological inputs; weaker hardening story.
- **Fix recommendation:** After generation, run the same sanitation pipeline used for canvas JSON load (at least for `data` and `headers`, and cap event/label lengths).

4) **Unknown trigger events are not editable without implicit mutation/data loss risk**
- **Where:** `NodeConfigPanel` → `TriggerConfig` uses a `<select>` with fixed `HOOK_EVENTS`.
- **Scenario:** `compileCanvasToHooks` explicitly “preserves unknown events”, and `generateCanvasFromHooks` will create triggers for any event key found.
- **Impact:** A trigger node with an unknown `data.event` will not have a matching `<option>`, which can render oddly and encourages accidental overwrite to a supported event on any user interaction.
- **Fix recommendation:** include an `<option value={data.event}>` for unknown events (or switch to a combobox/text input).

---

### LOW

5) **Palette icons use HTML entities as strings (renders literal `&#9889;` instead of ⚡)**
- **Where:** `NodePalette` passes `icon="&#9889;"` into `<span>{icon}</span>`.
- **Fix:** pass actual characters (`'⚡'`, `'⚙'`) or render entities via a safer approach.

6) **Minor React controlled-input robustness**
- **Where:** `TriggerConfig` uses `value={data.event}`; if `event` is ever `undefined`, React warns about controlled/uncontrolled input.
- **Fix:** default `value={data.event ?? ''}` and include a placeholder option, or ensure `TriggerNodeData.event` is always initialized.

---

## Type soundness / compilation logic notes (no additional severity)
- `compileCanvasToHooks` is intentionally shallow (Trigger→Condition→Action and Trigger→Action). That’s consistent with connection rules; it safely ignores unreachable/orphan nodes.
- `result.hooks` is `Object.create(null)` which is good for key-safety when stringifying.
- Debounce/read-from-refs approach is generally sound for ensuring latest nodes/edges are compiled.
