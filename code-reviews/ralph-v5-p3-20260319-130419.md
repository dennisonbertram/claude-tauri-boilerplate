## Audit: Visual Hook Builder (React Flow canvas → Claude Code hooks JSON)

### CRITICAL

1) **Untrusted `hooksCanvasJson` is loaded into ReactFlow state without strict sanitization/whitelisting (prototype-pollution & object-shape injection risk)**
- **Where:** `HookCanvas.tsx` mount effect:
  - Nodes: `const sanitizedNodes = saved.nodes.map((n) => ({ ...n, data: stripDangerousKeys(n.data) }))`
  - Edges: `setEdges(saved.edges as CanvasEdge[])` (**no sanitization**)
- **Why critical:**
  - `isValidCanvasState()` only validates a few required fields; it **does not reject extra keys** on node/edge objects (including `__proto__`, `constructor`, `prototype`, etc.) and does not enforce “plain object”.
  - These node/edge objects are handed to a large third-party state machine (`@xyflow/react`). Any internal `Object.assign`, merge, or cloning logic can turn a crafted `__proto__` payload into prototype pollution or unpredictable behavior.
  - Current code only strips dangerous keys in **`node.data`**, not on the **node object itself**, **edge objects**, or nested objects.
- **Fix:**
  - When loading saved canvas JSON, **reconstruct** nodes/edges using a strict whitelist and safe objects:
    - `node = { id, type, position: {x,y}, data: sanitizeNodeData(...) }` (no spread of untrusted objects)
    - `edge = { id, source, target }` only
  - Drop any extra keys everywhere (node, position, edge, nested).
  - Prefer `Object.create(null)` when constructing maps/records meant to hold user keys.

### HIGH

1) **User edits can be lost due to debounced persistence being canceled on unmount (no flush)**
- **Where:** `HookCanvas.tsx`
  - `notifyChange` (500ms) and `saveLayout` (2000ms) are canceled in cleanup:
    ```ts
    useEffect(() => () => { notifyChange.cancel(); saveLayout.cancel(); }, ...)
    ```
- **Failure mode:** Add/edit nodes then quickly switch views/profile or unmount within 500ms → changes never reach `onChange()` → **silent data loss**.
- **Fix:** implement `debounce(...).flush()` or keep latest `nodes/edges` in refs and on unmount:
  - flush pending structural save (compile + canvas json), then flush layout.

2) **`isValidCanvasState()` does not enforce size limits on strings/numbers for saved canvas → potential UI freeze / memory blowup**
- **Where:** `HookCanvas.tsx` `isValidCanvasState()`
  - Checks node counts/edge counts, but **not** `event/matcher/command/prompt/url/description/label` lengths.
- **Impact:** A profile containing `hooksCanvasJson` with huge strings (even with <=200 nodes) can lock the renderer on load and during `getHooksFingerprint()` stringification.
- **Fix:** enforce max lengths during validation **and** sanitize by slicing on load (not only on drag-drop). Apply the same `MAX_STRING_LENGTH` caps to loaded node.data.

3) **Graph invariants are not validated on load (invalid connections, duplicates, multi-incoming actions) → miscompilation / nondeterminism**
- **Where:** `isValidCanvasState()` validates edges are not dangling, but does **not** enforce:
  - `VALID_CONNECTIONS` (e.g., action→anything, →trigger)
  - **single incoming edge to action** invariant (enforced in `onConnect`, but not in load validation)
  - duplicate edges (same source/target)
  - self-loops
- **Impact:** A crafted/legacy canvas state can compile inconsistently:
  - `compileCanvasToHooks()` only walks from triggers and assumes shape; invalid edges can cause dropped nodes/hooks or unstable grouping.
- **Fix:** on load, reject or normalize edges to match `connectionRules` and enforce “<=1 incoming edge per action” + dedupe edges.

### MEDIUM

1) **Opening Canvas view triggers `onChange()` (compile + rewrite) even without user edits**
- **Where:** `HookCanvas.tsx` structural `useEffect()` starts with empty nodes fingerprint, then after initial load fingerprint changes and schedules `notifyChange()`.
- **Impact:**
  - Rewrites `hooksJson` formatting/order immediately (and may restructure groups).
  - If user has confirmed unsupported types warning, simply opening canvas will drop unsupported hooks/fields even if they “did nothing” in canvas.
- **Fix:** suppress initial compile/write:
  - set `prevStructuralRef` after initial state load, or gate with a `hasHydrated` ref and only compile on user-initiated edits.

2) **Trigger event UI cannot represent unknown/custom events cleanly**
- **Where:** `NodeConfigPanel.tsx` trigger `<select>` options are only `HOOK_EVENTS`.
- **Impact:** If hooks JSON contains an unknown event, canvas can display it, but config select may show blank; user interaction can inadvertently change it to a supported event.
- **Fix:** include a fallback `<option value={data.event}>Custom: {data.event}</option>` when current value isn’t in `HOOK_EVENTS`.

3) **Compiler grouping differences for “no matcher” groups**
- **Where:** `compileCanvasToHooks()`:
  - Trigger→Action direct emits **one group per action**.
- **Generator:** `generateCanvasFromHooks()` creates multiple Trigger→Action edges from a single group’s hooks list.
- **Impact:** Round-trip changes JSON shape (even if behavior is likely equivalent today). Risky if hook runner ever adds group-level semantics.
- **Fix:** represent “no matcher group” as an explicit container node or encode grouping metadata; or compile Trigger→Action-direct into a single group containing all direct actions in deterministic order.

4) **Type soundness gaps due to broad casting & weak runtime checks**
- **Where:** multiple `as CanvasNode[]`, `as any`, `onUpdateNode(newData: Record<string, unknown>)`.
- **Impact:** Invalid node data (e.g., wrong `hookType`, unexpected nested objects) can enter state via loaded canvas JSON and propagate to compiler/generator.
- **Fix:** strengthen runtime schemas (zod/io-ts) for canvas state and for action node data; narrow `onUpdateNode` by node type.

---

### LOW

1) **`useCallback(debounce(...), deps)` pattern creates throwaway debounced functions each render**
- Not usually a correctness bug, but unnecessary allocations. Prefer `useMemo` for debounced function creation.

2) **`connectionRules.ts` comment mismatch**
- Says “Used as the isValidConnection prop”, but it’s only used inside `onConnect`. Minor documentation drift.

3) **Duplicate `DANGEROUS_KEYS` definitions**
- `HookCanvas.tsx` defines a global set and also a smaller one inside `isValidCanvasState()` loop. Minor maintainability issue.
