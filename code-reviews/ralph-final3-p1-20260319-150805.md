## HIGH — Prototype-chain property access → persistent UI crash (DoS) via malicious `hookType` in `hooksCanvasJson`
**Where**
- `src/components/agent-builder/panels/NodeConfigPanel.tsx`
  - `const ConfigComponent = ACTION_CONFIG_COMPONENTS[data.hookType];`
  - `ConfigComponent && <ConfigComponent ... />`
- `src/components/agent-builder/nodes/ActionNode.tsx`
  - `const style = TYPE_STYLES[data.hookType] ?? TYPE_STYLES.command;`

**Why it’s exploitable**
`hooksCanvasJson` is treated as untrusted input. `isValidCanvasState()` does **not** validate `data.hookType` at all, and `sanitizeNodeData()` only filters **keys**, not dangerous **values**.

Because `ACTION_CONFIG_COMPONENTS`, `ACTION_TITLES`, and `TYPE_STYLES` are plain objects (default `Object.prototype`), indexing them with attacker-controlled strings like:
- `"__proto__"`, `"constructor"`, `"toString"`, etc.

…returns prototype properties (e.g., `Object.prototype` or `Object` function), not `undefined`. That can lead to:
- React trying to render a non-component object/function as a component → runtime exception → Hooks UI/tab can crash.

**Minimal PoC payload (stored in profile as `hooksCanvasJson`)**
```json
{
  "nodes": [{
    "id": "a1",
    "type": "action",
    "position": { "x": 0, "y": 0 },
    "data": { "hookType": "__proto__", "label": "boom", "prompt": "x" }
  }],
  "edges": []
}
```
Load canvas, click the node → `NodeConfigPanel` attempts to render `Object.prototype` as a component → crash.

**Impact**
- Persistent denial-of-service on opening/editing certain profiles (can be used as a “profile bricker”).
- In a Tauri context, repeated crashes can effectively block management/cleanup of malicious profiles.

**Fix**
- Treat `hookType` as untrusted and validate strictly everywhere (load + update):
  - allowlist: `command|http|prompt|agent`
- Replace “string-indexed plain objects” with safe dictionaries:
  - `Object.create(null)` maps, or `Map`
- Guard access:
  - `if (!Object.hasOwn(ACTION_CONFIG_COMPONENTS, hookType)) ...`

## HIGH — Resource exhaustion / UI freeze via huge `data` objects (key-count blowup) in `hooksCanvasJson`
**Where**
- `src/components/agent-builder/HookCanvas.tsx`
  - `stripDangerousKeys(n.data ?? {})` copies **all entries** with no cap
  - `sanitizeNodeData(...)` iterates **all entries** and preserves arbitrary primitive keys
  - `saveCanvas()` does `JSON.stringify` over full `data` objects

**Why it’s exploitable**
Even with node/edge count caps, a single node’s `data` object can contain **tens/hundreds of thousands of keys** (all primitive values), and it will:
1) pass `isValidCanvasState()` (only checks `data` is an object)
2) be copied twice (`stripDangerousKeys` + `sanitizeNodeData`)
3) be stored in state and repeatedly stringified during autosave

No limit exists for:
- number of keys in `data`
- overall `hooksCanvasJson` size
- overall parsed object complexity (beyond node/edge count)

**Impact**
- UI hang / memory blowup / potential app crash when opening a malicious profile.
- Persistent performance degradation because autosave repeatedly stringifies oversized state.

**Fix**
- Enforce:
  - maximum `hooksCanvasJson` length (bytes)
  - maximum number of keys per `data` object (e.g., 50)
  - strict allowlist of permitted keys per node type (recommended)
- Consider rejecting `hooksCanvasJson` if `JSON.parse` result exceeds budget.

## MEDIUM — Hidden HTTP `headers` supported by compiler but not editable/visible in UI → “stealth exfil” profiles
**Where**
- `src/lib/canvasCompiler.ts` supports `headers` and serializes them
- `src/lib/canvasGenerator.ts` parses `headers`
- `src/components/agent-builder/panels/NodeConfigPanel.tsx` **does not display/edit `headers` at all**

**Why it matters**
A shared/imported profile can include HTTP hooks with sensitive/malicious headers (e.g., `Authorization`, `X-Api-Key`) that:
- persist through canvas load/save cycles
- are not visible in the node preview (only shows method+URL)
- are not removable via UI (unless user switches to JSON and notices)

**Impact**
- Users can be tricked into running HTTP hooks with stealthy headers/tokens.
- Supply-chain style profile sharing becomes riskier because canvas view hides critical behavior.

**Fix**
- Either:
  - add headers UI (view/edit/delete), **or**
  - strip headers on load unless user explicitly enables “advanced fields”, **or**
  - show a prominent “Headers present” badge in ActionNode preview/config.

## MEDIUM — Unbounded `hooksJson` paste can cause repeated heavy parsing and generation (DoS-lite)
**Where**
- `src/components/agent-builder/tabs/HooksTab.tsx`
  - multiple `useMemo()` blocks repeatedly `JSON.parse(draft.hooksJson)`
- `src/lib/canvasGenerator.ts`
  - iterates groups/hooks until caps, but still can traverse large arrays until reaching limits

**Why it matters**
File import has a 500KB limit, but the textarea allows arbitrarily large input. A huge pasted JSON can:
- freeze the UI during repeated parsing/walking
- degrade responsiveness when toggling views (canvas generation)

**Fix**
- Impose a maximum length for `hooksJson` in UI (or at least before parsing).
- Parse once and share parsed result across memos (single parse + validation).
- Consider offloading parsing to a worker.

## LOW — Silent data loss: CRLF in URL is allowed in UI state but dropped during compilation
**Where**
- `src/components/agent-builder/HookCanvas.tsx`: `sanitizeNodeData` does not reject CRLF in `url`
- `src/lib/canvasCompiler.ts`: `buildHookEntry()` returns `null` if URL contains `\r`/`\n`

**Impact**
- User can input a URL containing newline; node looks configured, but hook disappears from compiled JSON on save.
- This is data corruption and can confuse audits (“why did my hook vanish?”).

**Fix**
- Validate URL at input time (or in `sanitizeNodeData`) and surface an error state.

## LOW — `hasUnsupportedHooks` can be bypassed by malformed non-iterable `group.hooks` (warning suppression → unexpected drops)
**Where**
- `src/components/agent-builder/tabs/HooksTab.tsx`
  - `for (const hook of group.hooks)` without checking `Array.isArray(group.hooks)`
  - any thrown error is caught → returns `false` (no warning)

**Impact**
- A malformed hooks JSON can suppress the “unsupported hooks” warning, yet canvas generation will skip hooks and subsequent save can overwrite/destroy data.

**Fix**
- Mirror `hasDangerousHooks` pattern: always `Array.isArray(group.hooks)` before iterating; treat malformed shapes as unsupported.
