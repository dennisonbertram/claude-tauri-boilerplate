# Documentation Index

Overview of the project documentation structure. Each folder has its own INDEX.md with file listings.

## Folders

| Folder | Description |
|--------|-------------|
| [research/](research/INDEX.md) | Research notes, external references, library evaluations |
| [design/](design/INDEX.md) | Design documents, wireframes, architecture diagrams |
| [exploration/](exploration/INDEX.md) | Exploratory investigations and app startup verification notes |
| [explorations/](explorations/INDEX.md) | Exploratory investigations, spikes, proof-of-concepts |
| [plans/](plans/INDEX.md) | Feature plans with checklists tracking progress |
| [investigations/](investigations/INDEX.md) | Deep-dive investigations into specific technologies or problems |
| [implementation/](implementation/INDEX.md) | Implementation notes, code change documentation |
| [testing/](testing/INDEX.md) | Test results and validation reports |
| [decisions/](decisions/INDEX.md) | Architecture decision records and technical rationale |
| [logs/](logs/INDEX.md) | Engineering log, observational log, systems log |
| [runbooks/](runbooks/INDEX.md) | Operational runbooks for testing, deployment, worktree workflow |
| [devlog/](devlog/INDEX.md) | Chronological developer log entries |
| [context/](context/INDEX.md) | Critical context documents for onboarding new agents |
