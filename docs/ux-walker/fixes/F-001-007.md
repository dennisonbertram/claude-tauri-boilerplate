# F-001-007: Show user-friendly error messages instead of raw stack traces

## What was wrong

The `chatError` memo in `ChatPage.tsx` passed raw `error.message` strings directly to the `ErrorBanner` component. When internal errors occurred (e.g., circular JSON serialization of React FiberNodes or HTMLElements), users saw unintelligible technical messages like:

> "Converting circular structure to JSON --> starting at object with constructor HTMLButtonElement..."

## What was changed

In the `chatError` useMemo block (`ChatPage.tsx` ~line 822):

1. Added `console.error('[ChatPage] Error details:', rawMsg)` so the full technical error is always logged for debugging.
2. Added detection for common technical/internal error patterns: `circular structure`, `fibernode`, `htmlelement`, `typeerror`, `referenceerror`, `syntaxerror`, `cannot read propert`.
3. When a technical error is detected, the user-facing message is replaced with: **"Something went wrong sending your message. Please try again."**
4. Non-technical errors (rate limits, auth, network) continue to show their original messages.
5. The existing retry mechanism via `ErrorBanner` remains fully functional.

## Build verification

The worktree has pre-existing build issues (missing `node_modules` for `@claude-tauri/shared`). The change itself is syntactically correct and type-safe -- it only adds string comparisons and a ternary assignment within the existing `useMemo` block.
