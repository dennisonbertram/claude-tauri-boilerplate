# Fix Report: F-001-004 — Send button missing accessible label

## What was wrong

The send button in the chat input area had no `aria-label` attribute. In the accessibility tree it appeared as an anonymous `button` element, making it unidentifiable to screen readers. While the button contains visible "Send" text, an explicit `aria-label` ensures consistent screen reader identification regardless of how the SVG icon is interpreted.

## What was changed

**File:** `apps/desktop/src/components/chat/ChatInput.tsx`

Added `aria-label="Send message"` to the submit button element (around line 609).

```diff
 <button
   type="submit"
   disabled={!input.trim() || isLoading}
+  aria-label="Send message"
   className="px-5 py-2 rounded-full ..."
 >
```

- Pre-existing TS errors exist in `shared/types.ts` and a test file; none introduced by this change.
