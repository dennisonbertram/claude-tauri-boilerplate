## Wave 1 — Critical Bugs

### Agent A: AgentProfileEditor.tsx

**File**: `apps/desktop/src/components/agent-builder/AgentProfileEditor.tsx`

1. **Delete Confirmation**: Currently `confirmDelete` state has NO cancel button, no timeout, no escape dismissal.
   - Add a "Cancel" button next to "Confirm Delete" button
   - Auto-reset `confirmDelete` to false after 3 seconds using `useEffect`
   - Add `useEffect` to listen for Escape key and reset `confirmDelete`

2. **Save Error Toast**: `handleSave` catch block only `console.error`. Add local `notification` state `{type: 'success'|'error', message: string}|null` with auto-dismiss (2s success, 4s error). Show as a small banner below header.

3. **Name Validation**: Save button should be disabled when `!draft.name?.trim()` in addition to `!hasChanges`. On save attempt with empty name, set notification error "Profile name is required."

4. **activeTab Reset**: Add `setActiveTab('general')` to the `useEffect` that resets draft on `profile.id` change. This fixes "New Profile Opens on Wrong Tab."

5. **Save Success**: When save succeeds (no throw), show notification `{type:'success', message:'Profile saved'}`.

6. **Cmd+S shortcut**: Add `useEffect` with `keydown` listener for `(e.metaKey || e.ctrlKey) && e.key === 's'` to trigger `handleSave`.

7. **Save button tooltip**: When disabled (no changes OR empty name), add `title="No unsaved changes"` or `title="Name is required"`.

8. **Tab labels**: Rename "Hooks" → "Automations", "MCP" → "Integrations" in the TABS array.

### Agent B: ToolsTab.tsx

**File**: `apps/desktop/src/components/agent-builder/tabs/ToolsTab.tsx`

1. **Tool Checklist**: Replace the two raw textareas (Allowed Tools, Disallowed Tools) with a visual checklist UI.
   - The Claude Code tools list is: `Read`, `Write`, `Edit`, `MultiEdit`, `Bash`, `Glob`, `Grep`, `LS`, `Task`, `WebFetch`, `WebSearch`, `TodoRead`, `TodoWrite`, `NotebookRead`, `NotebookEdit`, `ExitPlanMode`, `AskUserQuestion`
   - Each tool should be a checkbox row with icon and name
   - Three states per tool: "Allowed" (check), "Blocked" (x), "Default" (empty)
   - Keep a "Show raw" toggle to reveal the textarea for power users
   - The current `draft.allowedTools` and `draft.disallowedTools` arrays feed into this

2. **Allowed/Disallowed Logic**: Add a brief explanatory note: "If Allowed Tools is empty, all tools are available. Disallowed Tools override Allowed Tools."

3. **Container overflow**: Ensure the outer div has `max-w-full overflow-hidden` to prevent layout overflow.

### Agent C: GeneralTab.tsx

**File**: `apps/desktop/src/components/agent-builder/tabs/GeneralTab.tsx`

1. **Color Picker**: Replace `<input type="color">` with a simple hex color palette.
   - Keep the hex text input (it works fine)
   - Replace the native color swatch with a small popover grid of 12 preset colors
   - Clicking a swatch sets the hex value and closes popover
   - Use a state variable `colorPopoverOpen` to control popover
   - Preset colors: '#ef4444', '#f97316', '#eab308', '#22c55e', '#06b6d4', '#3b82f6', '#8b5cf6', '#ec4899', '#6b7280', '#1e293b', '#ffffff', '#000000'
   - The swatch button shows the current color as a filled circle, click opens popover

## Wave 2 — High Priority UX

### Agent D: AgentBuilderView.tsx

**File**: `apps/desktop/src/components/agent-builder/AgentBuilderView.tsx`

1. **Empty state CTA**: Add a "Create New Profile" button to the empty state. The existing handler `onCreateProfile` should be called.
   - Import `Button` from `@/components/ui/button`
   - Add `<Button onClick={() => { ... }} >Create New Profile</Button>` in the empty state div

2. **Navigation guard**: Before switching selected profile while editor has unsaved changes, show a confirmation.
   - Add `editorHasChanges` state to `AgentBuilderView`, passed down via a callback ref or lifted state
   - When `onSelectProfile` is called and `editorHasChanges` is true, show a `window.confirm("You have unsaved changes. Leave anyway?")` dialog

   **Simpler approach**: Pass `hasChanges` up from editor via a callback prop `onDirtyChange: (dirty: boolean) => void`. Track `isDirty` in `AgentBuilderView`. When selecting a different profile while dirty, prompt with `window.confirm`.

3. **Sidebar header title case**: Update `AgentProfileSidebar`'s header text - this is in `AgentProfileSidebar.tsx` (see Agent F below).
